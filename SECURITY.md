# Security Policy

This repository is in the foundation phase and does not yet publish a production-ready release.

Do not include private project files or sensitive technical details in public Issues or example templates. Use GitHub private reporting when available. Otherwise open a minimal Issue requesting a private communication channel.

Reports should identify the affected commit, template or parser area, reproduction steps, observed impact, and a sanitized example.

Template changes should be reviewed for unclear defaults, unexpected generated files, external dependencies, and misleading setup instructions.

Validated reports will be reviewed privately where practical. Public notes should explain affected content and corrections without exposing user information.
