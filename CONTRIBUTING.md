# Contributing

Read the README, About, and Roadmap before opening a change.

New templates should address a clear project type, use a consistent structure, include acceptance criteria, and remain understandable without a specific vendor tool.

Use an Issue for large template packs or format changes. Keep each pull request focused and include the problem, intended result, example input, expected output, validation steps, compatibility impact, and license information.

Use clear commit prefixes such as `feat:`, `fix:`, `docs:`, `test:`, `refactor:`, and `chore:`.

Contributors are responsible for correctness, licensing, and review of submitted material. Do not publish private information or material that you do not have permission to distribute.
